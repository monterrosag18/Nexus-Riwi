### 1. EL MOTOR TÁCTICO 3D (EL MAPA)
Ubicación: public/js/components/HexGrid.js

Líneas 9-16: Definición del Scope.
Aquí creaste las variables globales del motor: 'scene' (el universo), 'camera' (tus ojos), 'renderer' (el pincel), y 'raycaster' (el sensor que detecta dónde hace clic el usuario).

Línea 28: Función renderMap().
Es el punto de entrada. Aquí creas el contenedor HTML (div) que envuelve todo el mapa y le das un estilo de pantalla completa (Líneas 33-40) para que la experiencia sea inmersiva.

Líneas 100-115: Inicialización de la Escena.
Configuraste el universo 3D. Usaste un color de fondo negro puro (#000000) y agregaste luces ambientales para que los hexágonos brillen con ese estilo neón.

Línea 150 (aprox.): Generación de la Grilla Hexagonal.
¡ESTA ES LA MAGIA! Implementaste un bucle doble (q, r) que usa Coordenadas Axiales. En lugar de x/y (cuadrados), usaste matemática de hexágonos para que cada pieza encaje perfectamente sin dejar huecos.

Línea 190 (aprox.): Materiales y Shaders.
Elegiste materiales de tipo 'MeshBasicMaterial' con transparencia (transparent: true) y opacidad variable. Esto logra que los hexágonos parezcan hologramas digitales y no simples bloques sólidos.

Líneas 250+: Lógica de Interacción (Raycasting).
Programaste el sensor que detecta el mouse. Cuando el usuario hace clic, el 'raycaster' proyecta un rayo desde la cámara y te dice exactamente qué ID de hexágono fue tocado. Sin esto, el mapa sería una imagen estática.

### 2. EL SISTEMA DE CONQUISTA (BACKEND/API)
Ubicación: pages/api/territories/index.js

Línea 14: Lectura de Datos (GET).
Aquí conectaste el mapa con la base de datos Supabase. Esta línea hace un "select *" para traer el estado actual de quién es dueño de cada zona.

Línea 22: El Protocolo de Conquista (POST).
Este es el corazón de la competitividad. Cuando alguien intenta conquistar un sector, tú validas tres cosas críticas:
1. Línea 31: Verificas el Token de Seguridad (VerifyToken). Nadie puede hackear el mapa sin estar logueado.
2. Líneas 38-49: Validación de Adyacencia. ¡Tu mayor logro lógico! Programaste que el servidor revise si el hexágono objetivo está pegado a una base que ya pertenece al clan. Si no hay conexión, la petición se rechaza (Línea 90).
3. Líneas 103-106: Actualización Atómica. Si todo es correcto, cambias el dueño en la base de datos en tiempo real.

Líneas 110-131: Distribución de Puntos.
Programaste que el sistema premie al clan y al usuario con 100 puntos por conquista, y penalice al clan que perdió el territorio con -50 puntos. Esto crea una economía dinámica de guerra.

Línea 138: Broadcast Automático.
Hiciste que, tras cada conquista, el sistema envíe un mensaje automático al Chat informando a todos los jugadores qué sector fue asegurado y por quién.

### 3. NEURAL LINK (SISTEMA DE CHAT)
Ubicación: public/js/components/Chat.js

Líneas 43-51: Lógica de Interfaz.
Programaste los eventos de clic para minimizar y expandir el chat. Usas clases de CSS ('minimized' / 'expanded') para animar la ventana sin recargar la página.

Línea 53: El Evento 'Enter'.
Capturaste el teclado del usuario. Cuando presionan Enter, limpias el input (input.value = '') y disparas la función addMessage.

Línea 68: Función addMessage().
Aquí utilizas el 'store.js' (Línea 69) para saber quién está hablando. Lo más técnico aquí es la Línea 70-71: ¡Implementaste colores personalizados basados en el equipamiento del usuario! Si un usuario ganó un "Chat Color" en la tienda, tú aplicas ese color dinámicamente con un efecto de sombra (text-shadow).
